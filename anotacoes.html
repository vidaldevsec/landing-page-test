Tema central: Comandos Git Checkout, Revert e Reset

Git Checkout

  • Permite navegar entre commits e branches
  • git checkout [hash do commit]: navega para o commit especificado
  • Também possível navegar visualmente com extensão Git Graph

Diferença entre Git Revert e Git Reset

Git Revert

  • Cria um novo commit que desfaz alterações do commit anterior
  • Mantém o histórico de commits intacto
  • Útil para desfazer commits ruins mas manter registro do que aconteceu

Git Reset

  • Apaga commits até o commit especificado, removendo do histórico
  • Cuidado: pode remover commits importantes e dados de outros desenvolvedores
  • Geralmente é melhor usar Git Revert

Conclusão

  • Checkout: navegação
  • Revert: desfaz commit e mantém histórico
  • Reset: desfaz commit e apaga do histórico
  • Cuidado com Reset para não perder dados importantes
  • Melhor usar Revert na maioria dos casos

Esse resumo em forma de mapa mental cobre os principais tópicos e comandos abordados no vídeo. Fala sobre as diferenças e usos de Checkout, Revert e Reset, destacando pontos importantes como manter o histórico com Revert e o cuidado necessário com Reset.
