Trabalhando com branches no Git

O que é uma branch?

  • Ramificação do projeto, como um galho de uma árvore
  • Permite trabalhar em partes do projeto sem afetar o desenvolvimento principal
  • Cada desenvolvedor pode ter sua própria branch
  • Depois as branches são mescladas novamente na branch principal

Criando uma nova branch

  • git branch nome-da-branch: cria uma nova branch
  • git checkout nome-da-branch: navega para a branch criada
  • git checkout -b nome-da-branch: cria uma nova branch e já navega para ela

Visualizando branches

  • git branch: lista todas as branches locais

Apagando uma branch

  • git branch -d nome-da-branch: apaga branch se já tiver sido mesclada
  • git branch -D nome-da-branch: força a exclusão da branch

Trabalhando com branches

  • Cada branch segue seu próprio fluxo independente
  • Pode criar novas branches a partir de qualquer outra branch
  • Branches permitem trabalhar em paralelo sem afetar o código principal

Mesclando branches

  • Depois branches secundárias são mescladas novamente na principal
  • Permite juntar as funcionalidades desenvolvidas de volta ao projeto principal

Em resumo, branches são ramificações que permitem trabalhar de forma isolada em partes do projetosss sem afetar o desenvolvimento principal. Elas podem ser criadas, navegadas, visualizadas e apagadas conforme necessário.
